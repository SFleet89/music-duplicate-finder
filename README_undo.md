# Music Duplicate Undo

Restores files moved by `find_music_duplicates.py` back to their original locations. Reads any CSV report generated by the main script and reverses the moves — safely, with a dry-run option and full conflict handling.

Nothing is ever deleted. If a file can't be restored (missing, or something already exists at the destination), it is skipped with a clear explanation.

---

## Requirements

No additional dependencies — uses only Python standard library.

---

## Usage

```
python undo_duplicates.py <path_to_csv_report> [options]
```

**Options:**

| Flag | Description |
|---|---|
| `--dry-run` | Preview what would be restored without moving anything |
| `--list-categories` | Print all categories found in the CSV and exit |
| `--filter-category CAT` | Only restore files from a specific category |

**Examples:**

```
# Undo everything from a run
python undo_duplicates.py "C:\Music\Duplicates\music_report_20260302_120000.csv"

# Preview only
python undo_duplicates.py "C:\Music\Duplicates\music_report_20260302_120000.csv" --dry-run

# See what categories are in the report before filtering
python undo_duplicates.py "C:\Music\Duplicates\music_report_20260302_120000.csv" --list-categories

# Only undo exact matches
python undo_duplicates.py "C:\Music\Duplicates\music_report_20260302_120000.csv" --filter-category "Exact Match"

# Only undo duplicates
python undo_duplicates.py "C:\Music\Duplicates\music_report_20260302_120000.csv" --filter-category "DUPLICATE"

# Only undo better quality moves
python undo_duplicates.py "C:\Music\Duplicates\music_report_20260302_120000.csv" --filter-category "BETTER QUALITY"
```

---

## How it works

The script reads the CSV report and looks for rows where a file was actually moved (actions starting with `Moved to` or `Auto-moved`). Dry-run rows from the original run are automatically ignored.

For each moved file it:
1. Checks the file still exists at its destination
2. Checks nothing already exists at the original location
3. Recreates any missing parent folders
4. Moves the file back

If either check fails, the file is skipped and the reason is printed.

---

## Filter categories

Use `--list-categories` to see exactly what values are in a given report. Common values are:

| Category | Description |
|---|---|
| `Exact Match` | Files that were auto-moved as confirmed exact duplicates |
| `DUPLICATE` | Files selected for move during the Standard Duplicates review |
| `BETTER QUALITY` | Files selected for move during the Higher Quality review |

Older reports (from find_music_duplicates.py v3.0) may use `Standard Duplicate` or `Higher Quality Match` — these are also accepted.

---

## Compatibility

Works with CSV reports from `find_music_duplicates.py` v3.0 and later. The `Match Method` and `Why Not Exact` columns present in v3.1+ reports are intentionally ignored — they are informational only and not needed for undo.

---

## Notes

- The original CSV report is never modified
- Subfolders are recreated automatically if they no longer exist
- Running undo multiple times on the same report is safe — already-restored files will be skipped on subsequent runs
